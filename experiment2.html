Part 2 Instructions:


Now that you've learned about the blockworld language, it's time to create instructions for someone else to follow! Just like in part 1, there is a workspace and a starting pattern, but now you get to see the goal pattern, and need to come up with instructions in the blockworld language to help someone create it. Below is a schematic the the task screen, and the steps in each task. If you don't remember the language exactly, don't worry -- just do the best you can.
  1. Check out which blocks you have in the block section.
  2. Check out how is the task before and after solved.
  3. Write an instruction with everything you learned before using these keys.
  4. Press submit when you are done (Be careful and make the instruction understable and with logic using the vocabulary from the tasks you solved).

Buttons

Delete: Press the button to delete the last token.

Submit: Press the button when you have the desired block pattern to evaluate if its correct.

About the game:


In this game, I am investigating the difficulties and the ability of participants to communicate through different tasks with an artificial vocabulary.

As a player, you need to understand as much as you can the meaning of the instructions and solve the tasks remembering their meaning. Harder tasks contain a combination of different words from easier tasks. The more you understand each word, the easier it will be to solve the following tasks.

The objective of the game is to solve all tasks correctly understanding the words given and to create instructions for the next participant to solve the same tasks using your instructions. The instructions should be clear, short, and easy to understand for next participants.

Warning: Once you have done a task you can not go back to a previous task, you need to pay attention and try to get the idea of the instruction before going to the next task. You have a fixed number of attempts on each task, therefore, is not a good idea to try random sequences.
